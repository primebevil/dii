# 30-Day Research Plan

Week 1, Define. Nail the proposition: reliable floor, local-first, federated pods, non-commercial. Done; see Why_DII_Exists and the Architecture Overview.

Week 2, Research. Prior-art teardown of distributed compute, done in The Case Against DII. Next, one-page write-ups of why each prior project won or stalled.

Week 3, Prototype. Two-node, one-pod proof of concept: route a capability request from node A to node B and back. Measure the kill-criteria numbers, such as tokens per second over a residential link versus local, and overflow overhead.

Week 4, Pressure Test. Run the kill criteria, revise the architecture, and record decisions as ADRs.
